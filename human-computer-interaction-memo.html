인간과컴퓨터상호작용

Storytelling

  • 페르소나:
    • 같은 사람이 노트북을 사용할 때와 TV를 사용할 때 다른 인간이 된다.
    • TV 앞의 사람은 좀처럼 머리를 쓰지 않는다.

HCI와 UX

  • 사용자와 시스템 사이에 경험, 인터랙션, 인터페이스가 있음:
    • 경험은 목표, 인터랙션은 행동, 인터페이스는 도구.
    • 시스템에 맞닿는 면에 인터페이스가 있고, 이를 통해 인터랙션을 수행.
    • HCI의 목표는 사용자에게 최적의(optimal) 경험을 제공하는 것.
  • 영화 평론에서 말하는 최적의 경험:
    • 상업 영화의 세 가지 필수 경험 요소가 있음.
    • 사고적 수준(논리): 영화의 메시지가 관객에게 정확히 전달되어야.
    • 행동적 수준(속도): 스토리가 너무 빨라 이해할 수 없거나, 느려서 지루하지 않아야.
    • 감성적 수준(감정): 영화가 주는 감각적 자극이 주제에 적절해야.
  • HCI에서 말하는 ‘최적의 사용자 경험’:
    • 유용성: 사용자가 시스템을 이용해 목적을 효과적으로 달성할 수 있어야.
    • 사용성: 시스템을 사용하는 과정이 효율적이어야.
    • 감성: 사용자가 감정적으로 적절한 느낌을 받아야.

상호작용

  • 교수님 책상은 디제잉 대시보드 같다…
  • 상호작용이란:
    • 사회학적 정의: 둘 이상의 사람이 서로에게 영향을 주는 양방향적 관계.
    • 신방학적 정의: 사람과 메시지 사이에 일어나는 일련의 절차. (전통적으로는 일방적)
    • HCI: 디지털 아티팩트와 인간 사이에 발생하는 일련의 작용, 반작용 절차.
  • 컴퓨터와의 상호작용은 입출력 장치를 매개로 이뤄진다.
  • 인터페이스는 사람이 접하는 컴퓨터의 입출력 장치와 그 장치에 표현한 정보를 의미.
  • 상호작용 행위에는 지시, 대화, 탐색, 조종, 위임이 있음.
  • 상호작용성(interactivity):
    • “상호작용이 있다고 하려면 다음 같은 요소를 갖춰야 한다”
    • 상호성: 양방향 소통이 있어야.
    • 반응성
    • 신속성
    • 다중성: 다양한 매체(화면, 소리 등)를 통해 피드백을 제공해야.
    • 부가적으로 시간통제성, 내용통제성.
  • 상호작용의 종류(2x2 matrix):
    • 전송형: 상호작용의 내용과 전송을 모두 제공자가 주도. e.g., 방송
    • 등록형: 내용은 사용자가 만들지만, 상호작용의 전송 과정은 제공자가 주도. e.g., 유튜브
    • 문의형: 내용은 제공자가 만들지만, 전송(내용을 요청)은 사용자가 주도. e.g., VOD 서비스
    • 대화형: 내용과 전송을 모두 사용자가 주도. e.g., 메신저, SNS, P2P
    • 예시를 들긴 했지만 서비스에 따라 딱 떨어지게 1:1 분류할 수는 없음.

Usefulness

  • 유용성: 사용자가 시스템을 이용해 목적을 효과적으로 달성할 수 있어야.
  • 이미 사용자가 가지고 있는 습관과 지식을 활용하는 것이 가장 효과적.
    • 이를 BLT라고 한다(?): 빵, 양상추, 토마토…
    • BLT 샌드위치를 만들라고 했을 때 누구나 같은 것을 떠올린다.
    • 메타포를 쓰면 효과적. Desktop(바탕화면)은 실제 책상에 대한 메타포.
  • 사용자는 왜 기계를 사용하나? 가치를 실현하기 위해.
    • 기능적: 기능을 사용하기 위함.
    • 개인적: 개인적인 가치를 실현하기 위함. e.g., 명품, 커스텀 테마
    • 유희적: 즐기기 위함. e.g., 게임, 유튜브 시청
    • 사회적: 사회적인 가치를 실현하기 위함.
    • 부가적으로 금전적, 정언적(호기심, 얼리어답터), 조건적(특수한 상황, 소화기).
  • 사용자의 멘탈모델을 중심으로 유용한 시스템을 만든다.
    • 모델은 특정 목적으로 어떤 대상을 추상화한 결과. 복잡한 대상을 단순화한다.
    • 사람에 따라 멘탈모델이 모두 다르고, 불완전하다. 실제와 멘탈모델과의 괴리로 인해 사고가 발생하기도.
  • 행위이론:
    • 시스템을 원활히 사용하기 위해선 실행자와 평가자를 연결하는 매개체가 필요. 사용자가 시스템을 사용하면서 이러한 매개체를 만드는 과정을 설명하는 이론이 행위이론.
    • 7단계: 지각 > 해석 > 평가 > 목적 > 의도 > 행동 > 실행
    • 사용자의 컨텍스트를 파악하면 단계를 생략할 수 있음. 하지만 프라이버시 문제가 생길 수도.

Usability

  • 아주 중요한 챕터예요.
  • 사용성: 시스템을 사용하는 과정이 효율적이어야.
    • 좁은 의미:
    • 넓은 의미: 시스템을 통해 과업을 수행하는 과정에서 사용자가 느끼는 총체적인 품질.
  • 속성(차원):
    • 효율성: 주어진 과엽을 효율적으로 수행할 수 있는가?
      • 단축성: 사용자가 최대한 단순하게 원하는 과업을 마무리할 수 있는가?
      • 반응성: 반응속도가 일관성있게 빨라야. 시스템 딜레이, 네트워크 레이턴시 등.
        • 딜레이 상태를 이해하기 쉽게 보여줘야. e.g., 프로그래스 바
        • 오래 걸리는 작업은 시작 전에 미리 알려야.
    • 정확성: 사용자의 오류에 시스템이 대응하는 정도.
      • 사전방지성: 에러의 발생을 방지.
      • 오류발생감지성: 에러가 발생한 경우 빠르게 감지.
      • 오류회복성: 에러로부터 복구.
        • 후방오류회복: 오류 발생 이전 상태로 되돌린다.
        • 전방오류회복: 에러를 감수하고 다음 상태로 시스템을 진행. 사소한 오류인 경우에만.
    • 의미성: 사용자가 관심있는 정보가 제공되어야 한다.
      • 변화제시성: 시스템 상태가 변화했을 때 그 상태를 사용자가 감지할 수 있어야.
        • 즉시성: 변화가 발생한 즉시 사용자에게 변화된 상태를 제공.
        • 부가성: 사용자가 부가적인 명령을 내려야 변화된 상태를 제공.
      • 이해가능성: 전달된 정보를 통해 사용자가 시스템 상태를 이해할 수 있어야.
        • 가독성: 정보가 읽기 쉬움.
        • 논리성: 정보의 순서나 구조가 논리적임.
      • 학습성: 사용자가 시스템에 대한 지식을 쉽게 취득할 수 있다. (개인화와 상충)
        • 매뉴얼: 이해하기 쉬운 매뉴얼을 제공한다.
        • 적합성: 시스템의 종류에 맞는 목표를 제공한다.
        • 기억 가능성: 기억하기 쉽게 설계한다. e.g., 연말정산 서비스는 1년에 한 번 사용.
    • 유연성: 사용자가 원하는 작업을 시스템을 통해 원하는 방식으로 수행할 수 있어야.
      • 사용자 주도권: 사용자가 자신이 원하는대로 시스템과 상호작용할 수 있다.
        • 모드 최소화: 굳이 모드를 강요해야 한다면 현재 모드를 명확하게 알려야.
        • 과업 전이성: 사용자와 시스템이 통제권을 유연하게 주고받음. (e.g., 프린터)
      • 대체성: 사용자가 자신의 상황에 따라 적절한 방법을 선택할 수 있다.
      • 다중성: 사용자가 한번에 두 개 이상의 작업을 동시에 수행할 수 있다. e.g., 멀티스레딩
      • 개인화: 사용자의 특성에 따라 시스템의 특성을 변경할 수 있다. (학습성과 상충)
      • 연결성: 시스템 간의 연결을 쉽게 할 수 있다.
        • 연동성: 한 제품을 다른 제품과 쉽게 연결할 수 있다.
        • 호환성: 한 시스템의 기능을 다른 시스템에서 사용할 수 있다. (v1에서 만든걸 v2에서 사용할 수 있다면 backwards, 그 반대는 forwards)
    • 일관성: 시스템의 정보나 기능이 다른 대상과 비슷한 모습 또는 역할을 가지는가?

Usefulness & Usability Case Study: StarCraft

  • 프로덕트의 설계가 잘 되었다고 무조건 상업적인 성공을 거두진 않음.
  • 라프 코스터의 재미이론:
    • 재미: 어떤 문제를 정신적으로 해결했을 때 느끼는 즐거움.
    • 심미적 감상: 아름다운 것을 보거나 들었을 때 느끼는 즐거움.
    • 본능적 반응: 본능적인 반응을 그 자체만으로 즐겁지 않음. 도전을 성취했을 때 보상이 작동하면 즐거움을 느낌. e.g., 타이핑 작업은 재밌지 않지만, 타이핑 게임을 하면 즐거움.
    • 사회적 위치: 다른 사람과의 경쟁에서 느끼는 즐거움.
  • 게임 디자인의 핵심 요소:
    • Core mechanics
    • Storytelling and narrative:
      • 체스처럼 별로 중요하지 않은 경우도. 하지만 게임에 흥미를 갖게 할 수 있다.
      • 연속적인 구조(연속극)를 가질 수 있음.
    • Interactivity
  • 게임 디자인의 일반적인 요소:
    • 설정과 월드: 게임의 판타지를 구현.
    • 게임성(Game play):
      • 공정성: 실력 좋은 사용자를 이길 가능성이 높아야 한다. 실력자가 무조건 이기는게 아님.
      • 시드 마이어: 일련의 흥미로운 선택이 게임성이다.
      • 다양한 도전이 필요. 논리, 기억, 지능, 지식, 도덕 등 도전.
  • 게임은 사용자를 중독시킨다:
    • 사용자가 게임에 몰입할 수 있어야. 가짜라는 것을 잊게 됨.
    • 몰입을 깨는 요소: poor design, poor detail, poor harmony.
  • 하늘 아래 새로운 것 없다. 게임에도 원형이 있다:
    • Dune 2: 소설 <듄>을 배경으로 하는 RTS 게임. 스타크래프트에 영향.
    • 이후 C&C, 스타크래프트 프로토타입 버전 등이 만들어짐.
  • 오늘은 아예 스타를 하면서 보여줄게요:
    • 스타크래프트의 명령은 기본적으로 주어-동사-목적어의 조합:
      • SCV가(SCV 클릭) - 채굴한다(명령 패널의 채굴 아이콘 클릭) - 미네랄을(목표 미네랄 클릭)
      • 단축성: 동사를 생략, SCV가(SCV 클릭) - 미네랄을 채굴한다(목표 미네랄 우클릭)
    • 오류발생감지성: SCV로 미네랄을 공격한다면? 경고음이 발생하며 사용자에게 오류를 인지시킴.
    • 오류회복성: 건물을 짓다가 취소할 수 있음.
    • 반응성: 네트워크 레이턴시를 체감되지 않는 수준으로 관리. 실시간’처럼’ 보이게 구현.
    • 다중성: 명령이 큐에 들어감. 여러 유닛이 동시에 명령을 수행하기도.
  • 밸런스:
    • 공정한 게임이란, 실력 좋은 사람이 이길 가능성이 높은 게임.
    • 스타크래프트에서 랜덤하게 결정되는 건 스타팅 포인트 뿐. 승패를 뒤바꿀 랜덤은 없다.
    • 바둑은 먼저 시작한 사람이 유리하다. 나라마다 핸디캡이 다름.
    • Static balance:
      • 공격, 방어 등의 수치를 통한 밸런싱.
      • 스타크래프트는 버전 업데이트를 통해 해처리 건설에 필요한 미네랄을 낮춘 적이 있음.
      • Dominance strategy: 상대방의 전략과 상관없이 최선의 선택이 되는 전략.
    • Dynamic balance:
      • 컴퓨터와 대결하는데 사람이 잘하면 게임도중 컴퓨터의 실력이 높아짐.
      • 스타크래프트는 동적 밸런스를 별로 신경쓰지 않음. PvP가 메인 콘텐츠니까.
  • 4종류의 플레이어가 있음:
    • Killer: 플레이어 지향, 행동 선호, 빡겜러.
    • Socializers: 플레이어 지향, 인터랙션 선호, 친목러.
    • Achievers: 훨드 지향, 행동 선호.
    • Explorers: 훨드 지향, 인터랙션 선호, 월드 이곳저곳을 돌아다니며 구경하고 즐김.
    • 이 네 종류의 플레이어가 적절한 비율일 때 게임이 성공한다.
  • 월드 모델:
    • PvP 모델:
      • 어떻게 해야 난장판 만만투가 되지 않을까?
      • 결투가 가능한 공간을 한정…게임성이 저하. 다른 플레이어를 죽이면 아이디를 빨간색으로 보이게 만드는 패널티. 다른 플레이어들이 살인자를 응징한다. 만만투는 방지되고 게임성도 높아진다.

Affect

  • 감성은 결과론적인 성격이 강하다.
  • 감성에는 세 가지 측면이 있음:
    • 인상: 시스템이 어떤 느낌을 제공하는가?
    • 정서: 시스템이 어떤 감정을 제공하는가?
    • 개성: 시스템이 어떤 개성을 가졌는가?
  • 감정 분류와 Plutchik’s wheel of emotions 이야기…
  • 감성의 세부 구성 요소:
    • 인상: 실물에 의해 생체에 각인되는 변화, 대상에 대해 처음 마음에 나타나는 감정.
    • 정서: 감성 요소의 핵심. 비교적 단시간 동안 대상에 대해 갖는 감정.
    • 개성: 인상 중에서 특히 사람에 대해 느끼는 인상.
    • 정취: 강도는 약하지만 상대적으로 정서보다 오래 지속되는 감성.
  • E-Impression dimension:
    • 인상을 기본적 차원과 내용적 차원으로 분리.
    • 기본적: 의미가 추가되지 않은 인상.
      • BD: 밝은 - 어두운
      • CD: 차분한 - 강렬한
      • SV: 간결한 - 다채로운
    • 내용적: 의미가 추가된 인상, 시대에 따라 변화.
      • LP: 고급스러운 - 대중적인
      • CF: 고전적인 - 미래적인
      • OP: 희망적인 - 절망적인
  • 개성:
    • 원래 개성은 사람 또는 사람처럼 생각할 수 있는 대상에 대해 갖는 인상.
    • 하지만 제품이나 서비스, 브랜드도 개성을 가질 수 있음. 보통 의인화하여 표현.

User Analysis

  • 사용자 분석은 가장 중요한 단계.
    • 개발자도 사용자라는 인식. 자신의 경험이 보편적이라는 착각.
    • 사람들이 스스로의 필요를 모름. 자전거를 발명했는데 자전거 사용자를 조사할 수는 없음.
    • 분석할 시간과 비용이 부족함. 개인정보를 불법적으로 수집하기도.
  • 사용자는 두 종류로 나눌 수 있다:
    • 주 사용자: 실제로 시스템을 사용하는 사람.
      • 직접 시스템과 상호작용하는 사람들.
      • 실제 주 사용자를 분석하는 것이 중요.
    • 부 사용자: 주 사용자가 시스템을 어떻게 사용하는가에 영향을 주거나 받는 사람.
      • 실사용하지 않는 구매자(e.g., 학부모)
      • 주 사용자를 매니징하는 관리자.
  • 사용자에 대해 알고싶은 것:
    • 사용자의 숙련도, 행태적 특성, 개인적 특성.
    • 통계적 분석치가 중요한게 아님. (요상한 무성생식 외계인 예시…)

Task Analysis

  • 과업 분석의 목표는 사용자의 목적과 의도를 파악하는 것.
  • 시스템이 사용자의 의도와 목적에 부합하는 것은 유용성을 달성하기 위한 선결 조건.
  • 명시적 지식과 묵시적 지식:
    • 사용자는 매뉴얼, 규정과 같은 명시적 지식에 따라 시스템을 사용하지 않음.
    • 따라서 실제로 사용자가 어떻게 과업을 수행하는지 분석할 필요가 있다.
  • 계층적 과업 분석법:
    • 전통적으로 가장 많이 사용하는 방법.
    • 하나의 작업을 큰 태스크와 세부 태스크로 계층적 구분.
    • 분류를 어디까지 세분화할 것인지가 중요한 결정 사항.
  • 지식 기반 분석법:
    • 과업을 수행하기 위해 필요한 도구와 행위에 대한 지식에 기반.
    • 지식이 확보되지 않은 상태에서는 잘못된 분석을 할 수도 있음.
  • 시나리오 기반 분석법:
    • 시스템을 이용하는 사용자의 구체적인 경험을 순차적으로 기술.
    • 시나리오와 시스템 사이 연관성이 모호해짐. (시스템과 무관한 사용자의 모든 행동을 기술하니까.)
  • 시퀀스 모형 분석법:
    • 세부적인 과업을 실행해 나가는 과정을 순차적으로 기술.
    • 추상적이기 때문에 실제 상황을 이해하기 어려울 수 있음.
    • 결합 시퀀스 모형: 여러 사용자에게 과업을 수행하게 하고 일관된 시퀀스를 분류.
  • 직업 분석법:
    • 사용자가 일반적으로 하루나 한 달을 보내는 방식을 분석.
    • 제품이나 서비스 개발 초기 아이디어를 도출하려는 목적.
  • 작업흐름 분석법:
    • 전체적인 일을 수행하는 과정에서 누가 어떤 일을 하는지, 그 과정에 어떤 기능이 필요한지 분석.
    • 전체적인 흐름을 확인할 수 있음. 조직에는 적합하지만 개인을 대상으로는 부적절.
  • 맥락질문법

Context Analysis

  • 시스템을 디자인하는 과정에서 사용자가 어떤 환경에서 사용하는지가 중요.
  • 사용 맥락:
    • 좁은 의미로는 환경에 대한 설명, 넓게는 환경과 사용자의 상호작용 행위.
    • 물리적 맥락:
      • 사용자의 주관적 관념이 들어가지 않고 기계적으로 판단할 수 있는 맥락 요소.
      • 시간, 위치, 조명, 온도, 소음 등.
    • 사회적 맥락:
      • 사람과 사람간 관계적 내용을 파악해야 측정할 수 있는 맥락 요소.
      • 시간(업무 시간과 여가 시간), 위치(집/직장, 프라이버시), 조직 계층, 표준, 권력 등.
    • 문화적 맥락:
      • 한 집단에 속한 사람들을 다른 집단에 속한 사람들과 구분되게 하는 가치관.
      • 시간(과거/미래 지향적), 위치(권력 거리), 환경(불확실성 회피, 암시/명시)

Tech Analysis

  • 무조건 최고의 기술을 선택해야 하는 것은 아님.
    • 시스템이 갖춰지지 않은 상황에서는 사용자가 감당 가능한 기술의 한계가 없음.
    • 따라서 즉시 파괴적 혁신이 가능하다. e.g., 캄보디아는 QR 결제를 바로 수용함.
    • 이미 시스템이 갖춰진 경우에는 파괴적 혁신이 어려움. e.g., 한국은 여전히 카드 결제가 대세.
  • 디지털 기술의 분류:
    • 디지털 플랫폼: 여러 디지털 컴포넌트로 구성된 통합 시스템
    • 디지털 컴포넌트: 플랫폼을 이루는 단위 기술. e.g., 입력장치, 출력장치, …
    • 플랫폼이 능동적 vs 수동적, 도구적 vs 유희적, 개인용 vs 공유용으로 분류됨.
    • TV는 수동적+유희적+공유용 플랫폼. TV로 능동적, 도구적 과업을 수행하는 데 거부감이 있음.

Concept Design

  • 단순히 역량이 좋아야 하는게 아니라 많은 경험과 훈련이 있어야 좋은 컨셉을 만들 수 있음.
  • 브레인스토밍, 디자인씽킹, TRIZ 등 컨셉 디자인 방법론이 있음.
  • 기존 분석자료 검토 - 대안 컨셉 도출 - 최종 컨셉 선정 - 구체화
  • 기능집합 만들기:
    • 사용자 분석을 통해 페르소나 모형을 만들어야 한다.
    • 메타포와 시스템간의 일치도를 조사한다.
    • 메타포 모델을 작성한다.

Information Design

  • 사용자가 시스템의 정보 구조를 탐색하는 것을 내비게이션이라고 한다.
  • 정보 구조 디자인 단계:
    1. 사용자에게 의미있는 다양한 데이터를 수집하는 단계.
    2. 특정 기준에 따라 자료를 분류하는 단계.
    3. 정보의 구조를 결정하는 단계.
    4. 정보의 구조 사이를 이동하는 기본 방법을 정하는 단계.
    5. 내비게이션 지원 시스템을 디자인하는 단계.
  • 정보 구조 분류:
    • 네트워크 구조: 웹, HTTP
    • 대중분류구조
    • 변동구조
    • 혼합구조
  • 항해 지원 시스템

Interaction Design

  • 고객을 설득해서 생각을 바꾼다는 건 불가능한 일.
  • 간단하지만 유연한 인터랙션:
    • 상호작용의 단계 수가 적을수록 좋음.
    • 동시에 사용자가 시스템을 조정할 수 있는 여지를 줘야 한다.
  • 보편적이지만 예외적인 상황을 고려한 인터랙션:
    • 가장 가능성이 높은 경우를 대비해 인터랙션을 설계하지만, 예외적인 가능성도 고려해야.
    • 특히 사용자가 실수할 수 있는 케이스를 잘 대비해야.
  • 즉각적이지만 지체를 감안한 인터랙션.
  • 순응적이지만 대화가 가능한 인터랙션.
  • 부정적인 요소를 줄이기 위한 DO NOT 지침:
    • 사용자에게 무언가를 암기하도록 요구하지 말아야.
    • 사용자에게 세팅을 바꾸도록 강요하지 말아야.
    • 사용자로 하여금 오류를 범하게 유도하지 말아야.
    • 사용자로 하여금 쓸데없는 정보를 받게 하지 말아야.
    • 사용자에게 선택을 강요하지 말아야.
  • 긍정적인 요소를 증진하기 위한 DO 지침:
    • 사용자에게 관심을 가져야.
    • 한 발 먼저 내다봐야.
    • 상식적으로 행동해야.
    • 융퉁성있게 반응해야.
  • 스토리보드 기법:
    • 새로운 인터랙션을 설계하는 방식.
    • 어떤 시나리오에 대한 스토리 보드를 그리거나, 순서도를 그린다.

Interface Design

  • 기본적인 시각 디자인 요소:
    • 색(색상, 명도, 채도, 톤)
    • 모양(크기, 방위, 형태)
    • 레이아웃(정보의 양, 그룹, 정렬, 정보간 공간적 관계)
    • 타이포그래피
    • 그래픽(이미지, 다이어그램, 표 등)
    • 애니메이션
  • WIMP의 4요소: 윈도우, 포인터(커서), 메뉴, 아이콘(ISO 표준이 있음).
  • 균형성: 화면에서 보여지는 시각적 무게의 분포.
  • 대칭성: 대칭에서 편안함. 장점을 강조하고 싶다면 단점과 비대칭적으로 디자인해야.
  • 연속성: 자연스러운 시선의 흐름, 눈이 연속적 배치를 따라가게 됨.
  • 리듬감, 대비감, 비례감, 통일감, 단순미, 조밀도, 규칙성, 안정감
  • 일관성(적응성 및 예측 가능성), 다양성.
  • 인터페이스 디자인의 효과:
    • 유용성 측면:
      • 시각적 계층 구조를 이루는 화면 구성.
      • 대비, 전경과 배경의 구분, 기능별 배치, 색상과 톤을 이용한 계층 구조 형성 등.
    • 사용성 측면: 가독성. 색상과 폰트, 배치 등.
    • 감성 측면: 미적 인상(외부 자극으로 각인되긴 했지만, 정서로 관념화되지 않은 감성) 및 개성.

    • 메타포.
